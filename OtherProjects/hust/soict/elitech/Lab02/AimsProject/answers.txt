
1. If you create a constructor method to build a DVD by title then create a constructor method to build a DVD by category. Does JAVA allow you to do this?**

Java does not allow you to do this directly. Method overloading allows methods to have the same name but "different signatures, where the signature can differ by the number of input parameters, the type of input parameter(s), or both." Because both the `title` and `category` attributes are declared with the exact same data type (`String`), if you write two constructors that each take a single parameter for either `title` or `category`, they will share the exact same signature: `DigitalVideoDisc(String)`. Due to this duplicate signature, the Java compiler will throw an error.

2. Is JAVA a Pass by Value or a Pass by Reference programming language?**

Java is always a Pass by Value programming language.

3. After the call of swap(jungleDVD, cinderellaDVD) why does the title of these two objects still remain?**

The titles of the two objects remain the same because of Java's pass-by-value characteristic. The concept of "pass by value" means that "the method parameter values are copied to another variable and then the copied object is passed to the method." When you pass `jungleDVD` and `cinderellaDVD` into the `swap(Object o1, Object o2)` method, Java only creates copies of the reference addresses and assigns them to `o1` and `o2`. Changing the positions of `o1` and `o2` inside the method is merely swapping these local copies; it does not affect where the original reference variables outside the method are pointing.

4. After the call of changeTitle(jungleDVD, cinderellaDVD.getTitle()) why is the title of the JungleDVD changed?**

The title of `jungleDVD` is changed because, even though Java is pass-by-value, the value that is copied and passed into the method is actually the "memory address" pointing to the original object. When the local variable `dvd` executes the statement `dvd.setTitle(title);`, it uses this copied address to find the exact original disc in memory and changes its title. The fact that the method immediately follows this with the statement `dvd = new DigitalVideoDisc(oldTitle);` only makes the local variable `dvd` itself point to a completely new object; it cannot undo the title modification that was already successfully executed on the original object.
